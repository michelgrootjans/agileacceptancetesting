!1 User Management:
Users have always an unique username and email. 
When creating a user, that new user how no credit yet.

!1 Examples:
In the examples below 3 demo users are created in the test setup. Then the rules about the duplicate username and password are checked.

!2 The setup of this test will add three demo users:
!|Get all users|
|Name|Email|Credit Amount|
|michel|michel@ilean.be|0|
|pascal|pascal@ihc.be|0|
|W. Fall|wouter@BDUF.com|1000|

!2 Add one new valid user and two invalid users

!|Create users|
|Username|Password|email|Role|Result?|
|michel|iLean|michel@ilean.be|Developer|DuplicateUserName|
|peter|ihv|michel@ilean.be|Developer|DuplicateEmail|
|scott|tiger|scott@oracle.com|Developer|Success|

!2 Only the new user should appear in the list of users
!|Get all users|
|Name|Email|Credit Amount|
|michel|michel@ilean.be|0|
|pascal|pascal@ihc.be|0|
|scott|scott@oracle.com|0|
